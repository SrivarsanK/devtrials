# Plan 05-02: 72-Hour Forecast & Circuit Breaker Panel

## Objective

Build the predictive forecast and control interface enabling Insurer Operators to freeze policy issuance in high-risk zones, relying on LSTM-predicted liability models. Protect reinsurance agreements via auto-alerts.

## Tasks

- apps/insurer-dashboard/src/hooks/useForecasts.ts
- apps/insurer-dashboard/src/store/dashboardStore.ts
</read_first>

<action>
Create `apps/insurer-dashboard/src/components/insurer/ForecastPanel.tsx`.

It uses `useForecasts();` to handle mock loading states.
Extract `forecasts` (from local hook load), `pausedZones` and `toggleZonePause` from `useDashboardStore`.

Component features:
1. Reinsurance Alert (FCB-004): Sum all `predictedPayoutLakhs` across forecasts. If sum > 50 Lakhs, show a prominent red banner at the top of this panel: `🚨 Reinsurance Alert: Active liability projection exceeds ₹50 Cr Reinsurance threshold limits.`
2. Zone Grid/Table (FCB-001): List each layout with:
   - City & Zone names.
   - Dominant Trigger (text).
   - Expected Payout `₹xxL`.
   - Risk indicator Dot (`RED`/`AMBER`/`GREEN` based on `forecast.riskStatus`).
3. Circuit Breaker Control (FCB-002, FCB-003): Render a discrete button tracking locally paused status via Zustand's `pausedZones.has(forecast.zone)`. 
   - Label it UI-wise: If Paused, button is red, text says "Resume". If Active, button is `border-shield-amber text-shield-amber`, text says "Pause" (Allow toggling mostly for AMBER).
   - Connect click handler to `toggleZonePause(forecast.zone)`.
</action>
