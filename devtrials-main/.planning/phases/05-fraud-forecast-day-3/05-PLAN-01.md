# Plan 05-01: Fraud Detection Panel

## Objective

Implement the Fraud Detection Panel to give administrators insight into AI-flagged suspicious activities like GPS spoofing and anomalies, allowing them to manually override or triage flagged claims.

## Tasks

<task id="01.1" name="Implement resolveFraudAlert in Zustand Store">
<read_first>
- apps/insurer-dashboard/src/store/dashboardStore.ts
- apps/insurer-dashboard/src/types/index.ts (FraudAlert type)
</read_first>

<action>
Update `dashboardStore.ts`.

Add `resolveFraudAlert: (id: string, resolution: 'approve' | 'reject' | 'investigate') => void;` to the `DashboardState` interface logic.
Implement it:
```typescript
resolveFraudAlert: (id, resolution) =>
  set((state) => ({
    fraudAlerts: state.fraudAlerts.filter((f) => f.id !== id),
  })),
```
Ensure `getPendingFraudCount` returns correctly based on the filtered length.
</action>

<acceptance_criteria>
- `resolveFraudAlert` is implemented and available in the store
- Action removes the alert with the given `id` from the list
</acceptance_criteria>
</task>

<action>
Create `apps/insurer-dashboard/src/components/insurer/FraudPanel.tsx`.

It uses `useFraudAlerts();` to fetch the initial data (handles loading state smoothly).
Extract `fraudAlerts` and `resolveFraudAlert` from `useDashboardStore`.

Create a list view layout for the active alerts. Each card needs:
1. Header: `fraud.id` + `fraud.type` (e.g. GPS_SPOOF)
2. Score badge: A colored badge depending on `fraudScore` (Red if > 0.8, Amber if 0.5-0.8).
3. Meta info: Location (City, Zone) & Claim Amount.
4. Description mapping to `fraud.description`.
5. Footer: 3 small buttons mapping to the resolution action: "Approve" (green), "Investigate" (amber), "Reject" (red). Clicking calls `resolveFraudAlert`.

Header of panel shows `Pending: {fraudAlerts.length}` count.
</action>
